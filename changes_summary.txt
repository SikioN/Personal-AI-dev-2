Отчет об изменениях по задаче добавления темпоральной составляющей (s, p, o, t)

1. Что было сделано:
- Проведен анализ текущей структуры хранения знаний (триплетов).
- Реализовано "наивное" расширение триплета до квадруплета путем добавления четвертой координаты - времени (time).
- Обновлена логика индексации в векторной базе данных для поддержки временных узлов.
- Созданы блокноты для расчета метрик Knowledge Graph Completion (Hits@K, MRR) для обычного и темпорального графов.

2. Изменения в файлах:

Файл: src/utils/data_structs.py
- В dataclass Triplet добавлено поле `time: Node = None`.
- Метод `TripletCreator.create` теперь принимает необязательный аргумент `time`.
- Метод `TripletCreator.stringify` изменен так, чтобы при наличии временной составляющей она добавлялась в строковое представление (например, "2024: Субъект Предикат Объект"), что позволяет векторной модели учитывать время при поиске.
- Обновлена логика генерации ID триплета: теперь ID зависит от (s, p, o, t), что позволяет хранить разные факты для разных моментов времени.

Файл: src/kg_model/embeddings_model.py
- Метод `create_triplets` обновлен для поддержки новых темпоральных узлов. Если в триплете указано время, этот узел теперь также индексируется в векторном хранилище узлов (nodes collection) наряду с субъектом и объектом.

3. Добавленные файлы (метрики):

Файл: notebooks/metrics/read_metrics_existing.ipynb
- Содержит код для загрузки текущего графа и расчета метрик Hits@1, Hits@5 и MRR. Используется подход на основе векторного поиска: по паре (s, p) ищется истинный объект (o) среди всех узлов.

Файл: notebooks/metrics/read_metrics_temporal.ipynb
- Демонстрационный блокнот для работы с темпоральными данными.
- Создает тестовый набор квадруплетов (s, p, o, t).
- Индексирует их в отдельную временную БД.
- Рассчитывает метрики, учитывая временную составляющую в поисковом запросе, что позволяет валидировать работу системы с (s, p, o, t).

Файл: notebooks/metrics/wikidata_metrics.ipynb
- Специализированный блокнот на русском языке для работы с датасетом Wikidata Big.
- Реализует чтение данных в формате (s, r, o, start_t, end_t) и их маппинг в строковые Q/P идентификаторы.
- Содержит пошаговое описание алгоритма расчета метрик Hits@K и MRR для темпоральных графов.

Цель изменений:
Обеспечить возможность хранения и поиска знаний с привязкой ко времени без радикального изменения архитектуры (наивный подход через расширение метаданных и строкового представления триплета). Это позволяет использовать существующие механизмы RAG и графового поиска, учитывая временной контекст, в том числе на стандартных бенчмарках типа WikidataBig.
